Principles

Shot-first, job-aware, file-system honest.

The UI treats a shot as the workspace. Source videos, synced videos, calibration outputs, and motion capture outputs stay visible as durable state.

01 No capture affordances Browser camera recording is out of scope and must not appear in navigation or controls.
02 Every long task is a job Sync, calibration, and motion capture survive refresh through persisted job records.
03 FreeMoCap owns processing The web UI wraps upstream FreeMoCap and Skelly behavior instead of recreating it.
